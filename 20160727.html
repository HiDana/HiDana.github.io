Atom Snippets

atom code

前些日子受不了sublime一直跳出來付款的對話框，決定來試試看舅舅大推的atom

atom對於packages處理方式真的很棒很直覺化，而且大部分有支援sublime的其實在atom都有支援，介於此加上優美介面，立馬跳槽

跳槽有一項大工程就是搬移Snippets

在atom snippet的建立方式很不一樣，他屬於在一個文件建立所有snippet，不像sublime要各別存檔

首先，點開atom -> snippets

點擊後會自動打開一個叫snippets.cson的編輯檔案，裡面有簡單編輯檔案，檔案裡頭有簡單的解釋，其中有一段

‘.source.coffee’:          #指令適用的程式屬性
#‘Console log’:             #指令名稱
#‘prefix’: ‘log’            #指令快捷鍵
#‘body’: ‘console.log $1’   #指令內容物

程式屬性可參考下列

HTML: .text.html.basic
CSS: .source.css
SASS: .source.sass
JavaScript: .source.js
JSON: .source.json
PHP: .text.html.php
Java: .source.java
Ruby: .text.html.erb
Python: .source.python
plain text (including markdown): .text.plain

比如說我要加入一個屬於css的snippets,我就會加入下列幾行

# css
# check line
‘.source.css’:
 ‘css check line’:
 ‘prefix’: ‘lll’
 ‘body’: ‘’’
 border: 1px solid #a00;
‘’’

這樣我只要打lll就會出現border: 1px solid #a00了歐！！

參考網站 How to Create Your Own Snippets

如果，我今天要加入很多個css的snippets怎麼辦 不要傻傻的把上述打兩次歐，這樣atom只會吃到最後一個屬於css的snippets

# css
# check line
‘.source.css’:
 ‘css check line’:
 ‘prefix’: ‘lll’
 ‘body’: ‘’’
 border: 1px solid #a00;
‘’’
‘css check line2’:
 ‘prefix’: ‘ll2’
 ‘body’: ‘’’
 border: 1px solid #0a0;
‘’’

就如上述直接加在第一個‘.source.css’後面就好了！

好拉，經過以上的步驟，我相信已經能創造出簡易的snippetes了

但是要懶就要懶到極致，我想要打出snippetes後讓滑鼠游標直接落在我想要的位置怎麼辦？

標籤 content 中的游標順序，『$0』 代表最後的游標位置。所以如果你的標籤 content 中有 $0 $1 $2 $3，那麼你每按一次 tab 鍵，游標的順序會是 『$1 -> $2 -> $3 -> $0』

$0                 游標的地方不會有文字
${0:Hello_World}   游標的位置會有 hello_world 被反白

例如

# css
# check line
‘.source.css’:
 ‘css check line’:
 ‘prefix’: ‘lll’
 ‘body’: ‘’’
 border: 1px solid $0;
‘’’

如此一來，當我打出lll後，滑鼠游標就會自動位移到solid後囉！ ps.因為 『$』 符號，本來是內部用來設定游標位置的，所以要強制輸出 『$』 符號，務必添加斜線 『\』來跳脫字元，所以要寫成 『\$something』。
