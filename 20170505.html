Easy Use Postman

code postman api

最近開始在在網頁端越寫越後面

就會開始有一些關於api要串接測試的部分

這時候 postman 就會是你的好朋友好鄰居好麻吉

postman

postman有幾點好用的方法

下面簡單簡述一下

  • 各種 RESTful API 的 http 傳送

GET 、 POST 、 POST 、 DELETE等...的傳輸方式

  • Collections的管理方式

藉由不同 collection 去做 api 的管理

  • Environment 的設定

add environment 在畫面右上角的齒輪裡 manage environments

在做測試時，有時 push 到雲端後 同一路徑只是有不同的來源

比如說，針對新增一個user

local 端 ： localhost:3000/users

heroku 端： https://secure-stream-66664.herokuapp.com/users

後面路徑都是一樣的，因此我只要在環境上製作變數 url

修改原本的 url 改為 像 {{vue}} 的變數形式

如果要刪除的話就要針對那個id去做刪除， api 就會長得像 {{url}}/todos/:id

如果可以把 POST 時產生的 id 一併跟 environment 做整合的話，就不需要一直複製貼上了

可以在 POST 的 Tests 輸入下列兩行

var body = JSON.parse(responseBody)
postman.setEnvironmentVariable('todoId', body._id)

當 POST 時就會自動填入到 environment 裡

再把 DELETE 的 url 改成 {{url}}/todos/{{todoId}}

就會自動使用環境裡的 {{todoId}}

如果是 token 裡的 header 也可以這樣做

在 Tests 檔輸入

var token = postman.getResponseHeader('x-auth')
postman.setEnvironmentVariable('x-auth', token)

header 一樣可以用變數，這樣就會自動帶進去

可以點眼睛觀看目前自動帶入綁定環境的參數
