Github 專案管理

code github manage

在研究 github gantt 跟 issue 之間的關係時，發現 guthub 本身也有自己的專案管理 project 跟 milestone

github project

很像 trello 的 github project

先隨意新增一個 issue

點擊 create a project，新增一個 new project

點擊 add column，並新增一個 TO DO 的 column

隨意新增一個事項，可以把該事項轉換成 issue，並看到 issue 裡面看到剛剛轉換的新 issue 了

也可以按右上方的 add cards，把沒有加進 column 的 issue 拖拉進去

新增類別時也可以加入 type ，這樣當 issue 在哪個狀態時他就會自動移到該 column

當在 issue 裡 close issue 時

可以看到 issue 就自己跑道 complete(因為有增加偵測 close 的 type) 的欄位了

也可以點擊該 issue 後，直接點擊右邊底下的 close issue，就可以看到該 issue 自動跳到 complete 的 column

github milestone

在 issue 上方有個 milestone，新增一個新的 milestone，可以在該 milestone 編輯時間

在 新增 issue 時，就可以是哪個 milestone，就可以看到 剛剛那個 issue 已經到該 milestone 裡了

-

因為是研究 gantt 而發現 github 本身就有很棒的專案管理方式，gantt 也可以找很多開源的軟體，連接 github 就可以得到視覺化的 github issue gantt 圖了(e.g. gh-issues-gantt)
